Projet BackPack Hero

Le projet BackPack Hero a pour but de réaliser une application Java basée sur un jeu déjà existant : BackPack Hero. Le but du jeu est de traverser un donjon infesté d’ennemis avec des armes débloquées au fil de la partie. Les combats sont en tour par tour où le héros sélectionne une arme et nous voyons la prochaine action de chaque ennemi (se protéger, attaquer, soigner).


Pourquoi ce projet m’a permis de valider la compétence “Réaliser” ?


Élaborer et implémenter les spécifications fonctionnelles et non fonctionnelles à partir des exigences : Les consignes définies par les évaluateurs ont été respectées (le jeu est fonctionnel, la gestion de l’inventaire est correctement codée, les combats fonctionnent etc) et j’ai même rajouté du contenu supplémentaire dans le jeu.


Appliquer des principes d’accessibilité et d’ergonomie : Lors de la réalisation de cette application, j’ai fait en sorte que l’application soit la plus ergonomique possible, les différents boutons et interactions sont organisés comme dans le jeu original. Il y a du texte pour guider l’utilisateur à quasiment chaque interaction.


Adopter de bonnes pratiques de conception de programmation : Lors de ce projet, j’ai organisé le code en style MVC (Modèle, Vue, Contrôleur) qui est essentiel dans la programmation d’une application afin de mieux organiser le code. J’ai aussi dispersé le code de manière à ce qu’il n’y ait pas toutes les fonctions dans une seule classe. Mais surtout j’ai pris l’habitude d’automatiser les tests sur l’application.


Vérifier et valider la qualité de l’application par les tests : Des tests ont été effectués à chaque fois qu’une nouvelle fonctionnalité était codée mais aussi à la fin du projet où j’ai automatisé des tests afin de vérifier si l’application fonctionne convenablement.
